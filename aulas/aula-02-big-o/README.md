# Aula 02 — Big O e Eficiência

## Resultado da comparação

Para uma entrada com 1000 itens:

- O(1): 1 operação
- O(log n): 10 operações
- O(n): 1000 operações
- O(n²): 1.000.000 operações

## O que entendi

Big O não mede tempo em segundos.  
Big O mede como a quantidade de trabalho cresce conforme a entrada aumenta.

O(1) permanece constante.  
O(log n) cresce devagar porque divide o problema pela metade.  
O(n) cresce junto com a entrada.  
O(n²) cresce muito rápido porque usa loop dentro de loop.

## Conclusão

A eficiência de um algoritmo importa muito quando os dados aumentam.  
Um algoritmo que parece aceitável com poucos dados pode ficar muito lento com muitos dados.
